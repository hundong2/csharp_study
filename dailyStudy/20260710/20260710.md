## 1. Sequential Ring Buffer 수집

초당 수백만 건의 데이터 포인트를 받는 시스템에서는 매 입력마다 락을 잡는 구조가 바로 병목이 됩니다. `Interlocked.Add`로 쓰기 오프셋을 예약하면 여러 생산자가 서로 다른 위치를 확보할 수 있습니다.

- 코드: [1_SequentialRingIngest.csx](./1_SequentialRingIngest.csx)
- 핵심 문법: `struct`, `MemoryMarshal.Write`, `ReadOnlySpan<byte>`, `Interlocked.Add`
- 왜 필요한가: 시계열 수집은 순서와 처리량이 모두 중요하며, 링 버퍼는 고정 메모리로 지속 쓰기를 설계할 때 유용합니다.

## 2. 값 타입 기반 시계열 분석

원시 데이터 포인트를 매번 클래스로 감싸면 작은 할당이 폭발합니다. 값 타입과 정적 분석 메서드는 할당 없이 계산을 표현하기 좋습니다.

- 코드: [2_TimeSeriesExtension.csx](./2_TimeSeriesExtension.csx)
- 핵심 문법: `record struct`, `in` 매개변수, `double` 계산
- 왜 필요한가: 통계 계산은 데이터 개수만큼 반복되므로 객체 모델보다 메모리 접근 패턴이 더 중요합니다.

## 3. 체크포인트 CAS 가드

디스크 flush나 checkpoint는 동시에 여러 번 실행되면 오히려 저장소를 압박합니다. CAS 게이트로 하나의 flush만 허용하고 중복 요청은 빠르게 거절할 수 있습니다.

- 코드: [3_FieldCheckpointGuard.csx](./3_FieldCheckpointGuard.csx)
- 핵심 문법: `Volatile.Read`, `Interlocked.Exchange`, `CompareExchange`
- 왜 필요한가: 저장소 안정성은 처리량만큼 중요하며, 중복 flush는 장애를 키울 수 있습니다.

## 4. Striped Backoff

분산 캐시나 영속화 큐에 같은 키 요청이 몰릴 때 전체 시스템을 막기보다 stripe별로 중복 진입을 제한하는 방식이 유용합니다.

- 코드: [4_StripedBackoffCache.csx](./4_StripedBackoffCache.csx)
- 핵심 문법: 배열 stripe, 해시 분산, 원자적 진입/해제
- 왜 필요한가: 장애 방어는 전체 차단보다 작은 단위로 압력을 낮추는 쪽이 운영에 유리합니다.

## 5. 분기 친화 필터링

시계열 필터링은 대량 반복 작업입니다. 복잡한 분기를 한 곳에 몰아넣기보다 작고 인라인 가능한 메서드로 유지하면 JIT가 최적화하기 쉽습니다.

- 코드: [5_PgoBranchUnroll.csx](./5_PgoBranchUnroll.csx)
- 핵심 문법: `if`, `bool`, `AggressiveInlining`, 배열 루프
- 왜 필요한가: 핫 루프의 분기 예측 실패는 작은 비용처럼 보여도 수천만 번 반복되면 큰 지연이 됩니다.

## 정리

- 링 버퍼는 고정 메모리와 순차 쓰기를 결합하는 기본 패턴입니다.
- checkpoint와 persistence는 중복 실행을 제어해야 합니다.
- stripe와 backoff는 장애 상황에서 전체 시스템을 덜 흔들리게 합니다.
- JIT 친화 코드는 측정 가능한 성능 개선의 출발점입니다.
