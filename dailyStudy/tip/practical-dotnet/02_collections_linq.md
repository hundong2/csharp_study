# 02. Collections와 LINQ

실무 C#에서 가장 많이 만나는 자료구조는 `List<T>`, `Dictionary<TKey,TValue>`, `HashSet<T>`입니다. 데이터를 찾고, 필터링하고, 변환할 때 LINQ를 함께 사용합니다.

## 핵심 개념

- `List<T>`: 순서가 있는 목록입니다.
- `Dictionary<TKey,TValue>`: key로 value를 빠르게 찾습니다.
- `HashSet<T>`: 중복 없는 집합입니다.
- `Where`: 조건에 맞는 데이터만 고릅니다.
- `Select`: 데이터를 다른 형태로 바꿉니다.
- `GroupBy`: 값을 기준으로 묶습니다.

## 실무에서 왜 필요한가

API 응답, DB 조회 결과, 로그 이벤트, 메시지 큐 데이터는 대부분 컬렉션 형태로 처리합니다. 컬렉션과 LINQ를 모르면 서비스 코드 대부분을 읽기 어렵습니다.

## 주의점

LINQ는 편리하지만 무분별하게 중첩하면 성능과 가독성이 나빠질 수 있습니다. 핫패스에서는 단순 루프가 더 나을 때도 있습니다.
