## 1. 스트리밍 버퍼와 순차 쓰기

네트워크에서 들어온 데이터를 매번 새 배열로 복사하면 GC와 메모리 대역폭이 병목이 됩니다. `ArrayBufferWriter<byte>`는 필요한 만큼 버퍼를 얻고 실제 쓴 길이만 알리는 방식으로 스트리밍 구조를 연습하기 좋습니다.

- 코드: [1_DmaDataStreamer.csx](./1_DmaDataStreamer.csx)
- 핵심 문법: `struct`, `[StructLayout]`, `ArrayBufferWriter<byte>`, `Span<byte>`
- 왜 필요한가: 데이터 플랫폼에서는 복사 횟수와 할당 횟수가 처리량을 직접 결정합니다.

## 2. 메모리 매핑 데이터 표현

메모리 매핑 파일에서 읽은 원시 값은 래퍼 객체를 계속 만들기보다 값 타입과 정적 헬퍼로 다루는 편이 단순하고 빠를 수 있습니다.

- 코드: [2_ExtensionMemoryMap.csx](./2_ExtensionMemoryMap.csx)
- 핵심 문법: `record struct`, `in` 매개변수, 문자열 보간, `X16` 포맷
- 왜 필요한가: 대량 데이터 변환에서는 작은 객체 할당도 누적되면 큰 비용이 됩니다.

## 3. 백프레셔 게이트

하위 저장소나 시각화 엔진이 느려지면 상위 입력을 무한히 받으면 안 됩니다. CAS 기반 게이트로 한 번에 하나의 처리 흐름만 통과시키는 기본 구조를 만들 수 있습니다.

- 코드: [3_FieldBackpressureGuard.csx](./3_FieldBackpressureGuard.csx)
- 핵심 문법: `Interlocked.CompareExchange`, `Release`, 속성
- 왜 필요한가: 백프레셔는 장애 전파를 막는 데이터 파이프라인의 핵심 안전장치입니다.

## 4. Memory-Mapped File 캐시 세그먼트

큰 캐시 값을 일반 힙에 오래 보관하면 working set과 GC 부담이 커집니다. Memory-Mapped File은 큰 데이터 조각을 별도 메모리 영역처럼 다루는 실무 선택지입니다.

- 코드: [4_MmfCacheProtocols.csx](./4_MmfCacheProtocols.csx)
- 핵심 문법: `MemoryMappedFile`, `CreateViewAccessor`, `Encoding.UTF8`, 길이-prefix 프로토콜
- 왜 필요한가: 대형 데이터 세트는 힙 객체가 아니라 페이지와 세그먼트 단위로 설계해야 합니다.

## 5. 루프 인라인과 JIT 친화 코드

고성능 데이터 변환에서 수동으로 모든 메서드를 한 루프에 합치면 유지보수가 나빠집니다. 작고 순수한 메서드를 만들고 `AggressiveInlining` 힌트를 주면 JIT가 최적화하기 쉬워집니다.

- 코드: [5_PgoLoopVectorize.csx](./5_PgoLoopVectorize.csx)
- 핵심 문법: `double[]`, 루프, `MethodImplOptions.AggressiveInlining`
- 왜 필요한가: 성능과 가독성을 동시에 유지하려면 JIT가 이해하기 쉬운 코드 구조가 필요합니다.
