# 초보자 확인 단계

## 실행 확인

- [ ] `dotnet run --project .\src\CheckoutExercise`에서 문법과 주문 결과를 봤다.
- [ ] `--self-test`에서 `초보자 검증 통과 (4/4)`를 봤다.
- [ ] 숫자를 하나 바꾼 뒤 다시 빌드해 결과가 바뀌는 이유를 설명했다.

## 말로 답하기

1. `decimal`이 금액에 적합한 이유는 무엇인가?
2. `string?`와 `??`는 각각 무엇을 뜻하는가?
3. `CheckoutService`와 할인 정책의 책임은 어떻게 다른가?
4. 인터페이스가 할인 규칙 교체를 어떻게 돕는가?
5. 예상 가능한 입력 오류를 `Result`로 표현한 이유는 무엇인가?

## 짧은 복습 체크리스트

- [ ] 변수, 조건, 반복, 컬렉션, null 처리를 읽을 수 있다.
- [ ] `record`, 인터페이스, 생성자 주입을 한 문장씩 설명할 수 있다.
- [ ] Application Service, Strategy, Composition Root의 역할을 구분한다.
- [ ] 정상/경계/오류 사례를 각각 하나 이상 검증한다.
