# 초보자 확인 단계

코드를 보지 않고 말하거나 직접 해 보세요.

- [ ] `string`, `int`, `bool`, `enum`이 어떤 값을 담는지 설명한다.
- [ ] `string?`와 `??`가 필요한 이유를 설명한다.
- [ ] `if`와 `switch` 중 하나로 조건을 분기한다.
- [ ] `List<T>`와 `foreach`로 여러 값을 다룬다.
- [ ] `record`의 `with`가 새 값을 만든다는 것을 확인한다.
- [ ] 인터페이스는 계약, 클래스는 구현이라는 차이를 말한다.
- [ ] 서비스가 업무 흐름을, 저장소가 데이터 보관을 맡는 이유를 말한다.
- [ ] `async`/`await`가 오래 걸리는 작업을 기다릴 때 스레드를 불필요하게 막지 않도록 돕는다는 것을 설명한다.
- [ ] `dotnet run --project ./src/WorkBoardExercise -- --self-test` 결과가 4/4인지 확인한다.

실패하면 해당 검증 이름을 `Program.cs`에서 검색하고 입력·기대값·실제 계산을 차례로 읽으세요.
