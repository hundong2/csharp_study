# 초보자 확인 단계

코드를 닫고 먼저 답한 다음 `Program.cs`에서 근거를 찾으세요.

1. `string`과 `string?`는 무엇이 다른가요?
2. `Where`, `OrderBy`, `FirstOrDefault`는 각각 무엇을 하나요?
3. `await`를 쓰는 메서드의 반환 타입이 `Task<T>`인 이유는 무엇인가요?
4. 빈 제목과 null 인수 중 어느 쪽을 Result와 예외로 처리했으며, 왜 다른가요?
5. Repository와 Strategy가 각각 숨기는 변화는 무엇인가요?
6. Composition Root를 한곳에 두면 테스트와 운영 설정이 왜 쉬워지나요?
7. 메모리 저장소의 “조회 후 저장” 방식이 동시 요청에서 안전하지 않은 이유는 무엇인가요?

## 통과 기준

- 1~3번을 자기 말로 설명하고 자체 검증 4개를 모두 통과하면 기초 통과입니다.
- 4~6번을 코드 위치와 함께 설명하면 구조 이해 단계입니다.
- 7번에서 트랜잭션 또는 DB 고유 제약을 언급하면 운영 관점까지 도달했습니다.
