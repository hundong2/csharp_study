# 초보자 확인 단계

코드를 가리고 말하거나 적어 본 뒤 다시 확인하세요.

1. `var`와 `string?`는 각각 무엇을 뜻하나요?
2. `record`와 `with`가 변경 가능한 `class`보다 안전한 상황은 언제인가요?
3. `Any`가 표현하는 질문은 무엇인가요?
4. `async`/`await`가 필요한 대표적인 I/O 두 가지는 무엇인가요?
5. 이메일 오류에는 Result, null 인수에는 예외를 쓴 이유는 무엇인가요?
6. Application Service, Repository, Strategy의 책임을 한 문장씩 설명할 수 있나요?
7. Composition Root와 의존성 주입이 테스트를 쉽게 만드는 이유는 무엇인가요?
8. 메모리 중복 검사만으로 운영 환경의 동시 갱신을 막을 수 없는 이유는 무엇인가요?

6개 이상을 코드 없이 설명하고 자체 검증 4/4를 통과하면 오늘의 입문 단계를 완료한 것입니다.
