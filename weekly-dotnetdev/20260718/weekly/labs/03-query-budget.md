# 실습 3: N+1 쿼리 예산 감시

## 목표

N+1 문제는 ORM 기능 자체보다 "루프 안에서 관련 데이터를 다시 조회하는 구조"에서 먼저 발생합니다. 이 실습은 EF Core 없이 쿼리 카운터를 붙여 1+N 조회와 projection 조회의 차이를 보여 줍니다.

## 실행

```powershell
cd D:\workspace\csharp_study\weekly-dotnetdev\20260718\weekly
.\run.ps1 query-budget
```

## 관찰 포인트

- N+1 경로는 주문 목록 1회 조회 뒤 주문 수만큼 라인 조회를 반복합니다.
- projection 경로는 요약에 필요한 데이터를 한 번에 만들고 쿼리 예산을 넘지 않습니다.
- 통합 테스트에서는 실제 `DbCommandInterceptor` 같은 계측 지점에 쿼리 카운터를 붙일 수 있습니다.

## 확장 과제

- 주문 개수를 5개에서 100개로 늘려 쿼리 수가 어떻게 증가하는지 확인합니다.
- 예산을 `2`에서 `1`로 낮추고 실패 조건을 조정합니다.
- API 응답 직렬화 시 lazy loading이 발생하는 사례를 별도 테스트로 분리합니다.
